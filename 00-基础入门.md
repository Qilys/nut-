## 函数命名规则

### 函数名事件说明

```
// 函数名事件：人物状态、特效、ap、事件函数
// 某一个事件达成时会运行的函数
// 角色事件函数名：角色的事件不是状态的
```

### 常用事件函数前缀

- `onSetState_*` - 设置状态时执行
- `onAfterSetState_*` - 设置状态后执行
- `onEndState_*` - 状态结束时执行
- `onEndCurrentAni_*` - 当前动画结束时执行
- `onProc_*` - 状态持续过程中执行（组队双方运行）
- `onProcCon_*` - 状态持续过程中执行（仅自己运行）
- `onAttack_*` - 攻击敌人时执行
- `onBeforeAttack_*` - 攻击前执行
- `onAfterAttack_*` - 攻击后执行
- `checkExecutableSkill_*` - 判断技能是否可执行
- `checkCommandEnable_*` - 判断技能命令是否可用

**注意**: `*` 需要替换为实际的技能名称

## 数据类型转换

### 基本类型转换

```squirrel
// 转为浮点数
变量.tofloat()

// 转为整数型
变量.tointeger()

// 转为字符串
变量.tostring()
```

### 使用示例

```squirrel
local stateStr = tostring(obj.sq_GetState());    // 数值转字符串
local stateInt = tointeger(obj.sq_GetState());   // 转为整数类型
local stateFloat = tofloat(obj.sq_GetState());   // 转为浮点数类型
```

### 类型判断

```squirrel
Value = 123;
if (typeof Value == "integer") {
    local strValue2 = Value.tostring(); // 将整数变量转为字符串
}
```

### 字符串操作

```squirrel
local len = Value.len();                    // 字符串取长度
local son = Value.slice(i, i+1);            // 分割字符串
```

## 常用对象方法速查

### 状态相关

```squirrel
local state = obj.sq_GetState();            // 获取当前状态
local subState = obj.getSkillSubState();    // 获取技能子状态
local throwIndex = obj.getThrowIndex();     // 获取13状态使用的技能ID
local throwState = obj.getThrowState();     // 获取13状态下的子状态

local buffSkillIndex = obj.getBuffSkillIndex(); // 获取17状态使用的技能ID
```

### 坐标相关

```squirrel
obj.getXPos()      // 获得x轴
obj.getYPos()      // 获得y轴
obj.getZPos()      // 获得z轴
obj.setCurrentPos(x, y, z)  // 设定坐标（人物直接用会卡墙，怪物就不会）
obj.sq_SetfindNearLinearMovablePos(x, y, z)  // 不卡墙的人物坐标设置
```

### 方向相关

```squirrel
obj.setCurrentDirection(direction)  // 设定对象朝向
```

### 动画相关


### 移动相关

```squirrel

```

### 控制相关


## 函数合并方法

### 为什么需要合并函数

在程序中，一个唯一的函数名代表一个内存地址。如果函数名重复，后加载的同名函数会覆盖先加载的函数功能，导致某些功能莫名其妙地消失。

### 需要合并哪些函数

所有正常定义的函数，不管是回调函数还是普通函数，同名的话都会覆盖。**回调函数**是客户端在调用的，有冲突的话，影响的功能比较大。

### 如何检查函数冲突

按官方规范，回调函数一般都写在职业的common nut里面，比如鬼剑士的就在 `character/swordman/swordman_common.nut` 文件中。

**检查方法**：
1. 在对应职业的common nut文件中搜索函数名
2. 如果使用了多个nut脚本，可以使用文本编辑器（如notepad++）全局搜索函数名

### 合并函数示例

**冲突的函数**：

```squirrel
function aaa_Swordman(obj)
{
    111(obj);
}

function aaa_Swordman(obj)
{
    222(obj);
}
```

**合并后的函数**：

```squirrel
function aaa_Swordman(obj)
{
    111(obj);
    222(obj);
}
```

合并完后，就不需要上面的两个有冲突的回调函数了，直接删掉使用合并后的函数即可。

### 常见需要合并的回调函数

以下回调函数常见于各个职业的common nut中，如果有冲突需要合并：

- `onStartMap_Swordman` - 鬼剑士在副本每个房间开始时调用
- `drawCustomUI_Swordman` - 鬼剑士绘制近景层的回调函数（用于绘制血条、评分等UI）
- `onStartDungeon_职业名` - 进入副本时调用
- `procAppend_职业名` - 被动处理循环执行

### 合并函数注意事项

1. **保持函数逻辑**：合并时确保所有原有的逻辑都被保留
2. **参数一致**：合并的函数参数必须完全一致
3. **返回值处理**：如果有返回值，需要根据业务逻辑决定返回哪个值
4. **执行顺序**：合并后的函数执行顺序按照代码顺序执行
