## 怪物血量突破21亿限制

### 原理说明

DNF客户端是32位的，有符号整形的最大值为 `2^31 - 1 = 2147483647`（约21亿），最开头的数会被当作符号位，所以实际上只有31位用于表示血量。

要让怪物血量超过21亿，可以使用算法让血量"看起来"像突破了一样。

### 简单方法：掉血减半

使用AP的 `onSetHp` 函数，让怪物每次掉血时只减少一半。

```squirrel
function onSetHp_appendage_xxx(appendage, hp, attacker)
{
    local obj = appendage.getParent();
    if(!obj) return hp;
    
    local oldHp = obj.getHp();  // 获取当前HP
    local damage = oldHp - hp;  // 计算要减少的血量
    
    // 掉血减半
    local newHp = oldHp - (damage / 2);
    
    return newHp;
}
```

**说明**：
- 通过这种方式可以让怪物更抗揍
- 版本有加nut字体的话，不会影响显示的字体值
- 如果还不够用，可以改为 `(damage / 10)` 只减少十分之一或更少

### 高级方法：血量倍数算法

使用数组存储血量倍数，实现真正的血量翻倍。

```squirrel
function onSetHp_appendage_xxx(appendage, hp, attacker)
{
    local obj = appendage.getParent();
    if(!obj) return hp;
    
    // 获取数组：a=最大翻倍数, b=当前翻倍数, c=最大血量, d=当前血量
    local a = appendage.getVar("hpData").get_vector(0);  // 最大翻倍数
    local b = appendage.getVar("hpData").get_vector(1);  // 当前翻倍数
    local c = appendage.getVar("hpData").get_vector(2);  // 最大血量（真实）
    local d = appendage.getVar("hpData").get_vector(3);  // 当前血量（真实）
    
    local oldHp = obj.getHp();
    local damage = oldHp - hp;  // 计算伤害
    
    // 计算新的真实血量
    d = d - damage;
    
    // 如果血量超过上限，增加翻倍数
    if(d > c && b < a)
    {
        b = b + 1;
    }
    
    // 如果血量小于等于0，减少翻倍数
    while(d <= 0 && b > 0)
    {
        b = b - 1;
        d = d + c;
    }
    
    // 更新数组
    appendage.getVar("hpData").set_vector(1, b);
    appendage.getVar("hpData").set_vector(3, d);
    
    // 如果翻倍数用完了且血量为负，返回0（死亡）
    if(b <= 0 && d <= 0)
    {
        return 0;
    }
    
    // 返回计算后的血量
    return d;
}
```

**计算血量百分比**：

```squirrel
// 血量百分比 = (a / b) + (c / d) * (100 / b)
local hpPercent = (a.tofloat() / b.tofloat()) + 
                  (c.tofloat() / d.tofloat()) * (100.0 / b.tofloat());
```

**初始化示例**：

```squirrel
function onStart_appendage_xxx(appendage)
{
    local obj = appendage.getParent();
    if(!obj) return;
    
    // 初始化数组：最大翻倍数=10, 当前翻倍数=9, 最大血量=20000, 当前血量=20000
    appendage.getVar("hpData").clear_vector();
    appendage.getVar("hpData").push_vector(10);  // a
    appendage.getVar("hpData").push_vector(9);   // b = a - 1
    appendage.getVar("hpData").push_vector(20000);  // c
    appendage.getVar("hpData").push_vector(20000);  // d
}
```

**注意事项**：
1. 需要配合tbl文件把boss显示血量翻倍
2. 翻倍太多（10W、100W倍以上）后显示血量会变得不太正常，可能是nut精度问题
3. act的百分比全部失效，需要额外扩展对对象单独百分比回血的机制
4. 固定回血扣血是正常的，可以参与血量运算

## 红狗进图自动双刀

在 `onStartDungeon_Swordman` 函数中添加自动附加双刀的代码。

```squirrel
function onStartDungeon_Swordman(obj)
{
    if (!obj) return;
    
    // 判断是否为红狗（转职类型3）且学习了双刀技能（76）
    if(sq_getGrowType(obj) == 3 && obj.sq_GetSkillLevel(76) > 0)
    {
        obj.sq_IntVectClear();
        obj.sq_IntVectPush(0);        // throwState
        obj.sq_IntVectPush(0);        // throwType
        obj.sq_IntVectPush(76);       // throwIndex（技能ID）
        obj.sq_IntVectPush(0);        // throwChargeTime
        obj.sq_IntVectPush(0);        // throwShootTime
        obj.sq_IntVectPush(10);       // throwAnimationIndex
        obj.sq_IntVectPush(4);        // chargeSpeedType
        obj.sq_IntVectPush(4);        // throwShootSpeedType
        obj.sq_IntVectPush(10009);    // chargeSpeedValue
        obj.sq_IntVectPush(10009);    // throwShootSpeedValue
        obj.sq_IntVectPush(-1);       // personalCastRange
        obj.sq_AddSetStatePacket(STATE_THROW, STATE_PRIORITY_USER, true);
    }
}
```

**说明**：
- `sq_getGrowType(obj) == 3` 表示转职类型为3（红狗）
- `obj.sq_GetSkillLevel(76) > 0` 表示学习了技能76（双刀）
- 通过发送 `STATE_THROW` 状态来触发双刀效果

## 可选性柔化

可选性柔化允许在后摇期间可以施放其他技能，在刃影等职业中常用。

### 实现方法

在 `proc` 函数中写入技能取消逻辑：

```squirrel
function procAppend_职业名(obj)
{
    if(!obj) return;
    
    // 初始化全局取消标记
    if (obj.getVar("GLOBAL_PARAMETER_CANCEL").size_vector() <= 0)
    {
        obj.getVar("GLOBAL_PARAMETER_CANCEL").clear_vector();
        obj.getVar("GLOBAL_PARAMETER_CANCEL").push_vector(0);
    }

    local ani = sq_GetCurrentAnimation(obj);
    local state = sq_GetState(obj);
    
    // 在非战斗状态下重置标记
    if (state <= 20)
    {
        if (obj.getVar("GLOBAL_PARAMETER_CANCEL").get_vector(0) == 1)
        {
            obj.getVar("GLOBAL_PARAMETER_CANCEL").set_vector(0, 0);
        }
    }

    // 检测动画关键帧flag
    local isFlag = obj.sq_IsKeyFrameFlag(ani, 105);  // 105是自定义的flag ID
    if (isFlag)
        obj.getVar("GLOBAL_PARAMETER_CANCEL").set_vector(0, 1);

    // 如果标记为1，允许技能取消
    if (obj.getVar("GLOBAL_PARAMETER_CANCEL").get_vector(0) == 1)
    {
        // skill cancel 逻辑
        // 这里可以添加技能取消的判断和处理
    }
}
```

### ANI文件设置

在需要中断的动画（ani文件）中插入 `set flag`，flag ID要避免冲突（如使用114514等特殊数字）。

**重要**：
1. 技能设置状态成功后，记得重置 `obj.getVar("GLOBAL_PARAMETER_CANCEL")` 为0
2. flag ID要选择一个不会与其他功能冲突的数字
3. 老技能的ani只要在就能插入，很方便

## 其他技巧

### 怪物防御和抗性调整

如果只是增加怪物的防御和抗性，会导致固伤职业很尴尬。建议配合血量调整方法使用。

### 异常伤害频率调整

在异常伤害的AP中，可以通过调整时间间隔来改变异常伤害显示的频率：

```squirrel
// 异常伤害显示频率，数字越低显示的频率越高，帧数也越低
if(currTime - appendage.getVar("atkTime").getInt(0) > 300)
{
    // 300是时间间隔，可以调整这个值来改变频率
    appendage.getVar("atkTime").setInt(0, currTime);
    applyActiveDamage(appendage, newHpDamage, x, y, z);
}
```
